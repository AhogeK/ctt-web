## 执行流程

### 阶段 1: 读取核心配置文件

**必须**按顺序读取以下文件：

1. **行为约束** - @AGENTS.md（**优先读取，作为后续所有操作的准则**）
2. **项目记忆**（规则 1）:
   - @memory-bank/projectbrief.md
   - @memory-bank/techContext.md
   - @memory-bank/systemPatterns.md
   - @memory-bank/activeContext.md
   - @memory-bank/progress.md
3. **项目说明** - @README.md

**注意**: 如记忆文件不存在，按 AGENTS.md 模板创建。

### 阶段 2: 扫描实际代码结构

**必须使用 Glob 工具**扫描实际项目代码：

- 源代码：`src/**/*.ts`, `src/**/*.vue`
- 配置文件：`vite.config.ts`, `tsconfig*.json`, `package.json`
- 样式：`src/**/*.css`, `tailwind.config.*`
- 测试：`tests/**/*.spec.ts`, `src/**/*.test.ts`
- 其他：`.oxfmtrc`, `oxlint.json`, `eslint.config.*`

**禁止假设**: 不要假设代码"应该"存在或不存在，必须通过 Glob 确认。

### 阶段 3: 检查 .gitignore

**必须读取** @.gitignore 了解哪些文件被排除在版本控制外。

**关键原则**:

- `.env.local` 等本地配置通常被 ignore，这是正常的
- 不要对被 ignore 的文件大惊小怪或指出"问题"

### 阶段 4: 读取发现的代码文件

根据阶段 2 的扫描结果，读取实际存在的代码文件以了解：

- 项目结构（`src/` 目录布局）
- 依赖版本（**只读取，不质疑，不升级** - 遵守 AGENTS.md 规则 8）
- 已实现的功能模块

**版本处理规则**（AGENTS.md 规则 8）:

- ✅ "当前使用 Vue 3.5.13"（仅陈述事实）
- ❌ "Vue 3.6 还在 beta，建议降级到稳定版"（质疑用户决策，禁止！）

### 阶段 5: 确认边界情况

**必须读取** @memory-bank/activeContext.md 确认：

1. **当前正在处理的任务** - 避免偏离当前工作焦点
2. **已知错误/障碍** - 避免重复踩坑
3. **最近变更** - 了解最新进展
4. **下一步行动** - 明确接下来的任务

### 阶段 6: 输出摘要

读取完成后，输出：

```

- [列出发现的源文件和目录结构]

核心约束（按 AGENTS.md）:
- Git 操作需人工确认（规则 7）
- 技术决策需人工确认（规则 8）
- 自动更新记忆文件（规则 2）
- 只读取不猜测版本（规则 8）
```

## 核心原则

### 原则 1: 事实优先（了解上下文场景）

此命令的目标是**了解项目当前状态**，不是 review 代码质量。

- ✅ "当前 src/features/ 目录尚未建立，符合 scaffold 初始化阶段"
- ✅ "扫描到 3 个 .vue 文件，均为空壳组件"
- ❌ "组件结构混乱，缺少业务逻辑，需要重构"

**注意**: 如果用户要求 review 代码，那是另一个独立任务，应使用不同的评判标准。

### 原则 2: 版本只读

- 仅陈述实际使用的版本号
- 不质疑、不建议升级、不假设"应该"用什么版本
- 示例：
  - ✅ "当前使用 Vite 8.0.0"
  - ❌ "Vite 8 刚发布，稳定性存疑，建议使用 Vite 7"

### 原则 3: 结合上下文理解

- 参考 progress.md 中的开发阶段判断代码完备性
- 参考 activeContext.md 了解当前工作焦点
- 不要孤立地评判代码"缺了什么"

### 原则 4: 确认边界情况

启动时必须确认当前边界：

1. **当前任务边界**: 根据 activeContext.md 确认当前工作范围
2. **技术决策边界**: 根据 AGENTS.md 规则 8，涉及版本/架构变更必须人工确认
3. **Git 操作边界**: 根据 AGENTS.md 规则 7，所有 commit/push 必须获得明确授权
4. **AI 能力边界**: 根据 AGENTS.md 规则 9，不懂就问，优先使用现代 Vue 3 写法
5. **已知障碍**: 根据 activeContext.md 中的错误记录，避免重复踩坑

## 禁止事项

执行此命令时，**绝对禁止**：

1. **质疑版本**: "XX 版本不稳定/已过期，建议升级到 YY"
2. **主观评判**: 对代码现状做"好/坏"、"需要重构"等主观评价（**注意：review 任务除外**）
3. **大惊小怪**: 对被 .gitignore 的文件（如 `.env.local`）指出"问题"
4. **跳读文件**: 不读取 AGENTS.md 就执行操作
5. **不扫描代码**: 仅凭记忆文件假设代码状态

---

**区分场景**:

- `/boot` → 了解项目状态（客观陈述，不做评判）
- `/review` 或用户要求 review → 审查代码质量（可做价值判断）
