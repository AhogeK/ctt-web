description: 主动记忆 - 保存项目状态和变更到记忆库（防止 AI 幻觉遗忘）
---

**⚠️ 强制规则**: 此命令必须严格按照 AGENTS.md 执行，作为自动记忆的补充手段。

## 背景

虽然 AGENTS.md 规则 2 要求每次响应后自动评估更新记忆，但 AI 可能因上下文溢出、任务切换等原因遗漏重要信息。此命令用于*
*主动确认并保存**关键变更。

**与自动记忆的区别**:

- **自动记忆**: 每次响应后自动评估（规则 2）
- **主动记忆**: 用户或 AI 显式调用，确保重要信息不遗漏

## 执行流程

### 阶段 1: 检查需要记忆的内容

读取当前记忆文件，确认现有状态：

1. **读取记忆文件**:
    - @memory-bank/projectbrief.md
    - @memory-bank/techContext.md
    - @memory-bank/systemPatterns.md
    - @memory-bank/activeContext.md
    - @memory-bank/progress.md

2. **识别需要记录的新信息**:

   | 来源    | 需要记录的内容                               |
      |-------|---------------------------------------|
   | 代码修改  | 新增/修改的组件、composable、路由、API 调用         |
   | 决策确定  | 架构变更、状态管理策略、组件设计模式                    |
   | 错误/障碍 | 遇到的问题（如 Hydration mismatch、类型错误）、解决方案 |
   | 任务进度  | 完成的功能模块、进行中的任务、阻塞点                    |

### 阶段 2: 确定更新的记忆文件

根据信息类型选择目标文件：

| 信息类型                 | 目标记忆文件                            |
|----------------------|-----------------------------------|
| 当前工作焦点、待办事项、错误/障碍    | `activeContext.md`                |
| 任务完成状态、开发计划          | `progress.md`                     |
| 新的技术决策、组件设计模式、状态分层规范 | `systemPatterns.md`               |
| 依赖版本变化、新增工具          | `techContext.md`                  |
| 路由结构变更、新功能模块上线       | `README.md` + `systemPatterns.md` |

### 阶段 3: 执行记忆更新

**格式要求**:

```markdown
# 当前上下文

## 最近变更

- [YYYY-MM-DD] - [变更描述]

## 错误/障碍

- **问题描述**: [具体问题]
- **原因**: [分析]
- **解决方案**: [如何解决]

## 下一步行动

1. [待办 1]
2. [待办 2]
```

**时间获取**: 使用 `date "+%Y-%m-%d"` 获取当前日期

### 阶段 4: 输出记忆摘要

更新完成后，输出：

```
=== 记忆保存 ===

已更新:
- [文件 1]: [更新内容摘要]
- [文件 2]: [更新内容摘要]

保留未更新:
- [文件]: [原因，如"无新信息"]

下次 /boot 时，这些记忆将自动加载到上下文。
```

## 触发场景

建议在以下情况主动使用此命令：

1. **完成重要任务** - 功能模块实现、组件开发完成
2. **做出关键决策** - 状态管理策略调整、路由结构变更
3. **遇到并解决问题** - 类型错误、构建问题、运行时异常
4. **切换上下文** - 从一个 feature 转到另一个 feature
5. **长时间对话** - 每 10-15 轮对话后确认记忆
6. **用户明确要求** - 用户说"保存一下"、"记录当前状态"

## 禁止事项

1. **不要假设** - 必须先读取现有记忆文件再更新
2. **不要遗漏日期** - 必须包含 YYYY-MM-DD 格式的日期
3. **不要冗余** - 只记录新信息，不重复已有内容
4. **不要猜测** - 不确定的信息先询问用户确认

## 与 /boot 的关系

| 命令      | 目的     | 时机             |
|---------|--------|----------------|
| `/boot` | 读取项目状态 | 会话开始、切换上下文     |
| `/save` | 保存当前状态 | 完成任务、做出决策、遇到问题 |
